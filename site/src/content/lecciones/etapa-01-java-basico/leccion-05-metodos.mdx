## Qué vas a aprender

Después de esta lección serás capaz de declarar un método con parámetros y valor de retorno, llamarlo con distintos argumentos, y ofrecer variantes del mismo cálculo mediante sobrecarga.

## Por qué necesitas aprenderlo

Hasta ahora todo el código vivió dentro de `main`. Un sistema real necesita partir la lógica en piezas reutilizables — "calcular el total de una venta" es algo que vas a necesitar en más de un lugar.

## Qué debes saber antes

Tipos primitivos y operadores (lecciones anteriores de esta etapa).

## Recupera lo aprendido

¿Qué imprime `break` en un bucle, comparado con `continue`?

<Reveal titulo="Respuesta">
`break` corta el bucle por completo. `continue` solo salta el resto de la iteración actual y sigue con la próxima.
</Reveal>

## Problema

Necesitás calcular el total de una venta en varios lugares del programa: a veces con descuento, a veces sin. Copiar y pegar la misma cuenta en cada lugar es exactamente el tipo de duplicación que querés evitar.

## Modelo mental

```
calcularTotal(precio, cantidad)
      │              │
   parámetro      parámetro
      │              │
      └──────┬───────┘
             ▼
        cuerpo del método
             │
             ▼
        return resultado
```

## Explicación sencilla

Un método es un bloque de código con nombre que recibe datos de entrada (parámetros) y devuelve un resultado (`return`). Lo escribís una vez y lo llamás cuantas veces necesites, con distintos valores cada vez.

## Explicación técnica

Un método se declara con: modificador (`static` por ahora — vas a ver por qué cambia en la próxima etapa), tipo de retorno, nombre, y una lista de parámetros entre paréntesis. El cuerpo termina con `return valor;` si el tipo de retorno no es `void`.

**Sobrecarga (overloading):** podés declarar varios métodos con el **mismo nombre** siempre que tengan una lista de parámetros **distinta** (en cantidad o en tipo). Java decide cuál invocar según los argumentos que le pasás en cada llamada — esto se resuelve en tiempo de compilación, no es una decisión en tiempo de ejecución.

## Cómo funciona

1. `calcularTotal(1200.0, 5)` coincide con la versión de dos parámetros (`precio`, `cantidad`).
2. `calcularTotal(1200.0, 5, 0.10)` coincide con la versión de tres parámetros (agrega `descuento`) — es un método completamente distinto, aunque comparta nombre.
3. `hayStockSuficiente(10, 4)` devuelve un `boolean`, calculado y retornado en una sola expresión.

## Ejemplo mínimo

**Archivo:** `proyecto/etapa-01/Metodos.java`

```java
public class Metodos {

    static double calcularTotal(double precio, int cantidad) {
        return precio * cantidad;
    }

    static double calcularTotal(double precio, int cantidad, double descuento) {
        double total = precio * cantidad;
        return total - (total * descuento);
    }

    static boolean hayStockSuficiente(int stockActual, int cantidadPedida) {
        return stockActual >= cantidadPedida;
    }

    public static void main(String[] args) {
        double totalSinDescuento = calcularTotal(1200.0, 5);
        System.out.println("Total sin descuento: " + totalSinDescuento);

        double totalConDescuento = calcularTotal(1200.0, 5, 0.10);
        System.out.println("Total con descuento: " + totalConDescuento);

        boolean alcanza = hayStockSuficiente(10, 4);
        System.out.println("Hay stock suficiente: " + alcanza);
    }
}
```

## Predice

¿Cuál de los dos métodos `calcularTotal` se ejecuta cuando lo llamás con dos argumentos, y cuál con tres?

<Reveal titulo="Resultado real (compilado y ejecutado)">
```
Total sin descuento: 6000.0
Total con descuento: 5400.0
Hay stock suficiente: true
```
</Reveal>

## Explicación paso a paso

- `calcularTotal(1200.0, 5)` → `1200.0 * 5 = 6000.0`, sin restar nada.
- `calcularTotal(1200.0, 5, 0.10)` → mismo total `6000.0`, menos el 10% (`600.0`) = `5400.0`.
- Java eligió automáticamente cuál de los dos métodos `calcularTotal` llamar, según cuántos argumentos le pasaste — eso es sobrecarga en acción.

## Ejercicio guiado

Escribí una tercera sobrecarga de `calcularTotal` que además reciba un costo de envío fijo y lo sume al resultado.

<Reveal titulo="Pista 1">
La firma sería `calcularTotal(double precio, int cantidad, double descuento, double costoEnvio)` — cuatro parámetros, distinta de las otras dos.
</Reveal>

<Reveal titulo="Pista 2">
Podés llamar a la sobrecarga de tres parámetros DESDE la de cuatro, para no repetir la lógica del descuento.
</Reveal>

<Reveal titulo="Solución">
```java
static double calcularTotal(double precio, int cantidad, double descuento, double costoEnvio) {
    double totalConDescuento = calcularTotal(precio, cantidad, descuento);
    return totalConDescuento + costoEnvio;
}
```
</Reveal>

## Mini reto de debugging

```java
static int sumar(int a, int b) {
    int resultado = a + b;
}
```

¿Por qué no compila?

<Reveal titulo="Diagnóstico">
El método declara que devuelve `int`, pero el cuerpo nunca ejecuta un `return`. Calcular `resultado` no alcanza — hay que devolverlo explícitamente con `return resultado;`.
</Reveal>

## Resumen

- Un método agrupa lógica repetida: nombre, parámetros, tipo de retorno, cuerpo.
- Todo camino de ejecución de un método no-`void` debe terminar en `return`.
- La sobrecarga permite el mismo nombre con distinta lista de parámetros.
- Qué sobrecarga se ejecuta se decide en compilación, mirando la llamada, no en runtime.

## Modelo mental final

Cuando pienses en un método, imaginá una máquina expendedora: metés los parámetros (las monedas), la máquina hace su proceso interno, y te devuelve el resultado (`return`) — siempre el mismo proceso, con distintos insumos cada vez.
