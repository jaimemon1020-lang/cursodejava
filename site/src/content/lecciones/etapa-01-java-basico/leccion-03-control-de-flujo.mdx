## Qué vas a aprender

Después de esta lección serás capaz de decidir entre varios caminos posibles en un programa usando `if`/`else if`/`else`, y elegir entre un `switch` clásico o uno moderno según cuántas ramas y qué tan simples sean.

## Por qué necesitas aprenderlo

"Si hay stock, procesar el pedido; si no, avisar" es la lógica central de todo el sistema que vas a construir. Sin control de flujo, un programa solo puede ejecutar instrucciones en línea recta.

## Qué debes saber antes

## Problema

Necesitás decidir el costo de envío según el total de la compra: gratis por encima de cierto monto, con descuento en un rango medio, o costo completo por debajo. Eso son tres caminos posibles, no uno.

## Modelo mental

```
if (condición A)
   → camino A
else if (condición B)
   → camino B
else
   → camino C (ninguna de las anteriores)
```

## Explicación sencilla

`if` evalúa una condición: si es verdadera, ejecuta ese bloque y se salta el resto. `else if` da una segunda oportunidad si la primera fue falsa. `else` es el camino por defecto si ninguna condición se cumplió.

## Explicación técnica

Java evalúa las condiciones de un `if`/`else if`/`else` **en orden**, y ejecuta solo el primer bloque cuya condición sea verdadera — el resto se ignora, incluso si también serían verdaderos.

`switch` compara un mismo valor contra varios casos posibles. Existen dos formas:

- **Switch clásico** (statement): cada `case` necesita `break` explícito, o la ejecución "cae" al siguiente case (fall-through) — a veces querido, casi siempre un bug.
- **Switch moderno** (expression, con `->`): cada rama es independiente, no hay fall-through, y el `switch` puede devolver un valor directamente a una variable.

## Cómo funciona

1. Las tres ramas de envío se evalúan en orden: la primera condición verdadera "gana".
2. El switch clásico sobre `categoria` (un `char`) recorre los `case` hasta encontrar coincidencia, y se detiene en el `break`.
3. El switch moderno sobre `diaSemana` (un `int`) evalúa directamente el valor y devuelve el resultado de la rama que matchea, asignándolo a `nombreDia`.

## Ejemplo mínimo

**Archivo:** `proyecto/etapa-01/ControlFlujo.java`

```java
public class ControlFlujo {
    public static void main(String[] args) {
        double totalCompra = 45000.0;

        if (totalCompra >= 50000.0) {
            System.out.println("Envio gratis");
        } else if (totalCompra >= 20000.0) {
            System.out.println("Envio con 50% de descuento");
        } else {
            System.out.println("Envio con costo completo");
        }

        char categoria = 'P';
        String nombreCategoria;
        switch (categoria) {
            case 'P':
                nombreCategoria = "Periféricos";
                break;
            case 'M':
                nombreCategoria = "Monitores";
                break;
            case 'A':
                nombreCategoria = "Accesorios";
                break;
            default:
                nombreCategoria = "Sin categoria";
        }
        System.out.println("Categoria: " + nombreCategoria);

        int diaSemana = 3;
        String nombreDia = switch (diaSemana) {
            case 1 -> "Lunes";
            case 2 -> "Martes";
            case 3 -> "Miercoles";
            case 4 -> "Jueves";
            case 5 -> "Viernes";
            default -> "Fin de semana";
        };
        System.out.println("Dia: " + nombreDia);
    }
}
```

`totalCompra` vale `45000.0`. ¿Cuál de las tres ramas de envío se imprime?

<Reveal titulo="Resultado real (compilado y ejecutado)">
```
Envio con 50% de descuento
Categoria: Periféricos
Dia: Miercoles
```
</Reveal>

## Explicación paso a paso

- `45000.0 >= 50000.0` es falso → se evalúa la siguiente condición.
- `45000.0 >= 20000.0` es verdadero → se imprime "Envio con 50% de descuento" y el `else` se ignora.
- El switch clásico sobre `categoria = 'P'` entra directo al `case 'P':`, asigna `nombreCategoria` y corta con `break` — sin el `break`, seguiría ejecutando el `case 'M':` de abajo (fall-through).
- El switch moderno no necesita `break`: cada `->` es una rama independiente que devuelve un valor.

## Ejemplo aplicado al proyecto

Esta misma estructura —tres rangos de condición para decidir un costo— es la lógica que más adelante va a decidir el estado de un pedido (`PENDIENTE`, `CONFIRMADO`, `CANCELADO`), cuando trabajemos con `enum` en una lección posterior.

## Mini reto de debugging

```java
int nota = 8;
if (nota >= 6)
    System.out.println("Aprobado");
    System.out.println("Felicitaciones");
```

¿El segundo `System.out.println` se ejecuta siempre, o solo cuando `nota >= 6`?

<Reveal titulo="Diagnóstico">
Se ejecuta siempre. Sin llaves `{ }`, el `if` solo controla la línea inmediatamente siguiente ("Aprobado") — la segunda línea ya no forma parte del `if`, aunque la indentación sugiera lo contrario.
</Reveal>

## Resumen

- `if`/`else if`/`else` se evalúan en orden; gana la primera condición verdadera.
- El switch clásico necesita `break` o cae al siguiente case (fall-through).
- El switch moderno (`->`) no tiene fall-through y puede devolver un valor directo.
- Sin llaves `{ }`, un `if` solo controla la línea siguiente, no un bloque completo.

## Modelo mental final

Cuando pienses en `if`/`else if`/`else`, imaginá una fila de puertas: entrás por la primera que esté abierta, y las demás ni se revisan.
