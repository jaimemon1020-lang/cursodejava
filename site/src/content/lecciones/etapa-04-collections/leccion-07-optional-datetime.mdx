## Qué vas a aprender

Después de esta lección serás capaz de escribir una búsqueda que puede no encontrar nada SIN devolver `null`, usando `Optional`; y de guardar, comparar y formatear fechas con la API moderna de `java.time` en vez de la vieja `Date`.

## Por qué necesitas aprenderlo

`RepositorioEnMemoria` de la Etapa 4 Lección 3 solo sabe `agregar` y `listarTodos` — no tiene forma de buscar UN elemento específico. ## Problema

Necesitás un método `buscar` en `RepositorioEnMemoria<T>` que devuelva un producto por algún criterio. Si no lo encuentra, ¿qué devuelve? Si devolvés `null`, cada lugar que llame a `buscar` puede olvidarse de chequear y explotar con `NullPointerException`. Además, `Product` necesita una fecha de ingreso al inventario, y calcular "cuánto tiempo pasó" a mano con `long` de milisegundos es propenso a errores de zona horaria y de conteo.

## Modelo mental

```
Optional<T>: una caja que puede contener un valor T, o estar vacía.
             El compilador te OBLIGA a abrir la caja explícitamente
             antes de usar lo que hay adentro — nunca asumís que hay algo.

LocalDate:   una fecha (sin hora) inmutable, con métodos para
             comparar, sumar/restar días, y formatear.
```

## Explicación sencilla

`Optional<T>` es un contenedor que puede tener un valor adentro o estar vacío — como una caja cerrada que tenés que abrir para saber si hay algo, en vez de asumir que sí lo hay (que es lo que pasa cuando devolvés `null` y alguien lo usa sin chequear).

`LocalDate` representa una fecha (día, mes, año) sin hora ni zona horaria. Es inmutable: cada operación (`plusDays`, `minusMonths`) devuelve una fecha NUEVA, nunca modifica la original.

## Explicación técnica

`Optional.of(valor)` crea un Optional con un valor presente (nunca le pases `null` a `Optional.of`, usa `Optional.ofNullable` para eso). `Optional.empty()` crea uno vacío. Los métodos más usados:

- `isPresent()` / `isEmpty()`: chequean si hay valor, sin extraerlo.
- `get()`: extrae el valor, pero lanza `NoSuchElementException` si está vacío — úsalo solo después de chequear `isPresent()`, o mejor, evitalo.
- `orElse(valorPorDefecto)`: devuelve el valor si está presente, o el default si no.
- `orElseThrow()`: devuelve el valor, o lanza una excepción si está vacío (más explícito que `get()`).
- `ifPresent(consumer)`: ejecuta una acción SOLO si hay valor — sin necesidad de chequear antes.
- `map(funcion)`: transforma el valor de adentro SI está presente, devolviendo otro Optional; si está vacío, no hace nada y sigue vacío.

Para fechas, `java.time` (desde Java 8) reemplazó a la vieja `java.util.Date`, que era mutable y confusa. `LocalDate.now()` da la fecha actual del sistema. `Period.between(fecha1, fecha2)` calcula la diferencia entre dos fechas en años, meses y días. `DateTimeFormatter` controla cómo se muestra una fecha como texto (`dd/MM/yyyy`, por ejemplo).

## Cómo funciona

1. `RepositorioEnMemoria<T>.buscar(Predicate<T> criterio)` recorre los elementos y devuelve `Optional.of(elemento)` en cuanto el criterio da `true`, o `Optional.empty()` si termina el bucle sin encontrar nada.
2. `Predicate<T>` es una interfaz funcional del propio `java.util.function` — recibe un `T` y devuelve `boolean`. Se le pasa como lambda: `p -> p.getNombre().equals("Teclado")`.
3. `Product` ahora guarda `fechaIngreso` (un `LocalDate`), asignada automáticamente a `LocalDate.now()` en el constructor.
4. `Period.between(fechaInicio, fechaFin)` calcula la diferencia como años/meses/días, no como un número plano de milisegundos.

## Ejemplo mínimo

**Fragmento actualizado de `proyecto/etapa-04/RepositorioEnMemoria.java`:**

```java
import java.util.Optional;
import java.util.function.Predicate;

public Optional<T> buscar(Predicate<T> criterio) {
    for (T elemento : this.elementos) {
        if (criterio.test(elemento)) {
            return Optional.of(elemento);
        }
    }
    return Optional.empty();
}
```

**Archivo:** `proyecto/etapa-04/OptionalFechasDemo.java`

```java
import java.time.LocalDate;
import java.time.Period;
import java.time.format.DateTimeFormatter;
import java.util.Optional;

public class OptionalFechasDemo {
    public static void main(String[] args) {
        RepositorioEnMemoria<Product> repositorio = new RepositorioEnMemoria<>();
        repositorio.agregar(new Product("Teclado mecanico", 25000.0, 15));
        repositorio.agregar(new Product("Mouse inalambrico", 8000.0, 40));

        Optional<Product> encontrado = repositorio.buscar(p -> p.getNombre().equals("Teclado mecanico"));
        if (encontrado.isPresent()) {
            System.out.println("Encontrado: " + encontrado.get().getNombre());
        }

        Optional<Product> noEncontrado = repositorio.buscar(p -> p.getNombre().equals("Monitor 4K"));
        String nombreOMensaje = noEncontrado.map(Product::getNombre).orElse("Producto no disponible");
        System.out.println(nombreOMensaje);

        noEncontrado.ifPresent(p -> System.out.println("Esto nunca se imprime"));

        Product teclado = encontrado.orElseThrow();
        DateTimeFormatter formato = DateTimeFormatter.ofPattern("dd/MM/yyyy");
        System.out.println("Fecha de ingreso: " + teclado.getFechaIngreso().format(formato));

        LocalDate fechaCompra = LocalDate.of(2025, 1, 15);
        LocalDate hoy = LocalDate.now();
        Period antiguedad = Period.between(fechaCompra, hoy);
        System.out.println("Antiguedad desde " + fechaCompra.format(formato) + ": "
                + antiguedad.getYears() + " anios, " + antiguedad.getMonths() + " meses, " + antiguedad.getDays() + " dias");
    }
}
```

## Predice

`noEncontrado` es un `Optional` vacío. `noEncontrado.ifPresent(p -> System.out.println("Esto nunca se imprime"))` — ¿esa línea imprime algo?

<Reveal titulo="Resultado real (compilado y ejecutado)">
```
Encontrado: Teclado mecanico
Producto no disponible
Fecha de ingreso: 18/07/2026
Antiguedad desde 15/01/2025: 1 anios, 6 meses, 3 dias
```
</Reveal>

## Explicación paso a paso

- `ifPresent` NO imprime nada para `noEncontrado`: el `Optional` está vacío, así que el lambda que le pasaste ni siquiera se ejecuta — por eso "Esto nunca se imprime" no aparece en la salida.
- `noEncontrado.map(Product::getNombre)` tampoco ejecuta nada (sigue vacío), así que `.orElse("Producto no disponible")` es lo que finalmente se imprime.
- `encontrado.orElseThrow()` extrae el `Product` de forma segura porque ya confirmaste que está presente — si `encontrado` hubiera estado vacío, esta línea habría lanzado `NoSuchElementException`.
- La fecha de ingreso muestra HOY porque `Product` la asigna con `LocalDate.now()` en el constructor — vas a ver una fecha distinta cada día que ejecutes el programa.
- `Period.between` calcula la diferencia real en años/meses/días, no un número plano de días — mucho más legible que hacer la cuenta a mano.

## Ejemplo aplicado al proyecto

Con `buscar` devolviendo `Optional<T>`, cualquier código futuro que busque un producto por nombre, código, o cualquier criterio queda obligado por el compilador a manejar el caso "no encontrado" — no hay forma de olvidarse, como sí pasaba (y pasa constantemente en proyectos reales) con `null`.

## Resumen

- `Optional<T>` representa un valor que puede faltar, obligando a manejar ese caso explícitamente.
- `orElse`, `orElseThrow`, `ifPresent` y `map` cubren los casos de uso más comunes sin necesitar `get()` directo.
- `LocalDate`, `Period` y `DateTimeFormatter` reemplazan a la vieja `Date`: son inmutables, más seguros y más legibles.
- `Predicate<T>` permite escribir un `buscar` genérico y reutilizable, en vez de un método fijo por cada criterio de búsqueda.

## Modelo mental final

Pensá en `Optional` como una caja cerrada con o sin premio adentro: no podés usar el premio sin abrir la caja primero (chequear o usar `map`/`orElse`) — y eso es exactamente lo que evita que agarres "nada" pensando que es "algo".
