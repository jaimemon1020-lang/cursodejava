## Qué vas a aprender

Después de esta lección serás capaz de reemplazar un array de tamaño fijo por una `List`, agregar y quitar elementos dinámicamente, y explicar por qué `Order` necesitaba esto desde la Etapa 3.

## Por qué necesitas aprenderlo

En un e-commerce real, un cliente agrega productos al carrito de a uno, sin saber de antemano cuántos van a ser. Tu `Order` de la Etapa 3 exigía pasar TODOS los `OrderItem` en el constructor, de una — eso no representa cómo se arma un pedido en la vida real.

## Qué debes saber antes

Arrays (Etapa 1) y el `Order`/`OrderItem` de la Etapa 3.

## Recupera lo aprendido

¿Por qué `OrderItem`, y no `Order`, calcula el subtotal de una línea de pedido?

<Reveal titulo="Respuesta">
Porque OrderItem es el único objeto que tiene el producto y la cantidad juntos — esos dos datos son los que hacen falta para el cálculo.
</Reveal>

## Problema

Con `OrderItem[] items` (Etapa 3), había que saber cuántos ítems iba a tener el pedido ANTES de crear el array. Un cliente real agrega productos al carrito uno por uno, sin ese número fijado de antemano.

## Explicación sencilla

Una `List` es como un array que puede crecer y achicarse: le agregás elementos con `.add()`, se los quitás con `.remove()`, y consultás cuántos tiene con `.size()` (no `.length`, esa es de arrays).

## Explicación técnica

`List` es una **interfaz** (Java Collections Framework) — describe el contrato "colección ordenada de elementos". `ArrayList` es la implementación más común: internamente usa un array que Java redimensiona automáticamente cuando hace falta más espacio, así que vos nunca gestionás ese detalle.

`List<OrderItem>` usa **Generics**: el `<OrderItem>` le dice al compilador "esta lista solo va a contener objetos `OrderItem`" — si intentás `list.add("texto")`, no compila. Esto te da seguridad de tipos sin tener que castear manualmente al leer elementos, como hacías con arrays de `Object` en otros lenguajes. Vas a profundizar Generics (tus propias clases genéricas) en la próxima lección.

Operaciones principales de `List`:
- `add(elemento)` — agrega al final.
- `remove(elemento)` — quita la primera coincidencia (por `equals`).
- `get(indice)` — accede por posición, igual que un array.
- `size()` — cantidad actual de elementos (no `length`).
- `contains(elemento)` — busca si existe, usando `equals`.

## Cómo funciona

1. `Order` ahora guarda un `List<OrderItem>`, inicializado vacío en el constructor con `new ArrayList<>()`.
2. `agregarItem(item)` llama a `items.add(item)` — el pedido crece de a uno, como en la vida real.
3. `calcularTotal()` sigue recorriendo con `for-each`, exactamente igual que con el array de la Etapa 3 — `List` también es iterable.
4. `nombresProductos.remove("Mouse inalambrico")` demuestra algo que un array NUNCA podría hacer directamente: achicarse quitando un elemento específico.

## Ejemplo mínimo

**Archivo:** `proyecto/etapa-04/Order.java`

```java
import java.util.ArrayList;
import java.util.List;

public class Order {

    private Cliente cliente;
    private List<OrderItem> items;

    Order(Cliente cliente) {
        this.cliente = cliente;
        this.items = new ArrayList<>();
    }

    public void agregarItem(OrderItem item) {
        this.items.add(item);
    }

    public double calcularTotal() {
        double total = 0;
        for (OrderItem item : items) {
            total += item.calcularSubtotal();
        }
        return total;
    }

    public int cantidadDeItems() {
        return this.items.size();
    }

    public void describir() {
        System.out.println("Pedido de: " + this.cliente.getNombre());
        for (OrderItem item : items) {
            System.out.println("  - " + item.getNombreProducto() + " x" + item.getCantidad()
                + " = " + item.calcularSubtotal());
        }
        System.out.println("Total: " + this.calcularTotal());
    }
}
```

## Predice

`Order` ahora se crea SIN items (`new Order(cliente)`), y se les agrega después con `agregarItem`. ¿Cuánto va a devolver `cantidadDeItems()` justo después de crear el pedido, antes de agregar nada?

## Explicación paso a paso

- `new ArrayList<>()` empieza vacía — `cantidadDeItems()` daría `0` antes de cualquier `agregarItem`.
- Cada `agregarItem` hace crecer la lista en uno, sin que el código de `Order` se preocupe por redimensionar nada.
- `nombresProductos.remove("Mouse inalambrico")` quita ESE elemento puntual (comparando con `equals`) — el resto de la lista se reacomoda sola.
- Imprimir una `List` directamente (`System.out.println(nombresProductos)`) muestra sus elementos entre corchetes — eso es el `toString()` que `ArrayList` ya trae incorporado.

## Resumen

- `List` es una colección de tamaño dinámico; `ArrayList` es su implementación más común.
- `add`, `remove`, `get`, `size`, `contains` son sus operaciones principales.
- `size()` (método) reemplaza a `length` (campo) — sintaxis distinta, mismo concepto.
- `List<Tipo>` usa Generics para garantizar que solo contenga elementos de ese tipo.

## Modelo mental final

Cuando pienses en `List` vs. array, imaginá un carrito de supermercado (crece a medida que agregás productos) contra una caja de huevos (siempre el mismo número de espacios, ni uno más).
