## Qué vas a aprender

Después de esta lección serás capaz de declarar variables en Java eligiendo el tipo correcto (primitivo o referencia), explicar la diferencia entre ambos, y usar `final` para modelar una constante — todo aplicado a datos reales de un producto del sistema de pedidos.

## Por qué necesitas aprenderlo

Todo lo que vas a construir en este curso — desde un `Product` con precio y stock, hasta una entidad JPA más adelante — empieza por declarar correctamente sus datos. Un error de tipo acá se arrastra hasta el final.

## Problema

Necesitás representar los datos de un producto: su nombre, cuánto cuesta, cuánto stock queda, si está disponible. Cada uno de esos datos es de una naturaleza distinta — texto, número con decimales, número entero, verdadero/falso. Java te obliga a decirle explícitamente qué tipo es cada uno.

## Modelo mental

```
VARIABLE
   │
   ├── TIPO PRIMITIVO (int, double, boolean, char...)
   │        el valor vive directamente en esa casilla de memoria
   │
   └── REFERENCIA (String, y cualquier objeto)
            la casilla guarda una dirección; el valor real vive en otro lado
```

## Explicación sencilla

Un tipo primitivo es una cajita que contiene el valor mismo: si declarás `int stock = 15;`, en esa variable hay literalmente un 15. Un `String`, en cambio, es una referencia: la variable apunta a un objeto `String` que vive en otro lugar de la memoria.

## Explicación técnica

Java tiene 8 tipos primitivos: `byte`, `short`, `int`, `long` (enteros), `float`, `double` (decimales), `boolean` (verdadero/falso) y `char` (un carácter). Todo lo demás — `String`, arrays, cualquier clase — es un **tipo referencia**: la variable almacena una referencia (dirección) al objeto real, no el objeto en sí.

Una **constante** se declara con el modificador `final`: una vez asignada, no puede reasignarse. Por convención se escribe en mayúsculas (`IVA`, no `iva`).

## Cómo funciona

1. `int stock = 15;` reserva espacio para un entero y guarda 15 directamente ahí.
2. `String nombre = "Teclado mecanico";` reserva espacio para una referencia; el texto real vive como objeto `String` en el heap.
3. `final double IVA = 0.21;` reserva espacio para un `double`; el compilador rechaza cualquier intento posterior de reasignarlo.

## Ejemplo mínimo

**Archivo:** `proyecto/etapa-01/DatosProducto.java`

```java
public class DatosProducto {
    public static void main(String[] args) {
        final double IVA = 0.21;

        String nombre = "Teclado mecanico";
        int stock = 15;
        double precio = 25000.0;
        boolean disponible = stock > 0;
        char categoria = 'P';

        double precioConIva = precio + (precio * IVA);

        System.out.println("Producto: " + nombre);
        System.out.println("Categoria: " + categoria);
        System.out.println("Stock: " + stock);
        System.out.println("Disponible: " + disponible);
        System.out.println("Precio sin IVA: " + precio);
        System.out.println("Precio con IVA: " + precioConIva);
    }
}
```

## Explicación paso a paso

- `final double IVA = 0.21;` — una constante, no cambia en todo el programa.
- `String nombre = "..."` — tipo referencia, entre comillas dobles.
- `int stock = 15;` — entero, sin decimales.
- `double precio = 25000.0;` — decimal, aunque termine en `.0`.
- `boolean disponible = stock > 0;` — el resultado de una comparación, no un literal.
- `char categoria = 'P';` — un solo carácter, entre comillas **simples** (no dobles).

## Mini reto de debugging

```java
char categoria = "P";
```

¿Por qué no compila?

<Reveal titulo="Diagnóstico">
`char` espera comillas **simples** (`'P'`). Con comillas dobles, `"P"` es un `String` de un carácter, no un `char` — son tipos distintos e incompatibles.
</Reveal>

## Resumen

- Java tiene 8 tipos primitivos: el valor vive directamente en la variable.
- `String` (y cualquier objeto) es una referencia: la variable apunta al valor real.
- `final` convierte una variable en constante — no puede reasignarse.
- El tipo se elige según la naturaleza real del dato, no por conveniencia.
- `char` usa comillas simples; `String` usa comillas dobles.
- Un error de tipo se detecta en **compilación**, antes de ejecutar nada.

## Modelo mental final

Cuando pienses en una variable primitiva, imaginá una caja con el valor adentro. Cuando pienses en un `String` (o cualquier objeto), imaginá una caja con una flecha que apunta a otra parte de la memoria donde vive el valor real.
