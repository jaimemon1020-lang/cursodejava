## Qué vas a aprender

Después de esta lección serás capaz de crear un array, acceder y modificar sus elementos por índice, recorrerlo con los bucles que ya conocés, y usar un array de dos dimensiones cuando el dato lo pida.

## Por qué necesitas aprenderlo

Ya usaste un array de pasada en la lección de bucles. Ahora formalizás: cómo se crea, qué pasa si accedés fuera de rango, y por qué su tamaño es fijo — algo que te va a importar cuando en la Etapa 4 lo compares con `List` (tamaño dinámico).

## Qué debes saber antes

Bucles `for`/`for-each` (lección anterior de esta etapa) y métodos.

## Recupera lo aprendido

¿Qué diferencia hay entre `for` clásico y `for-each`?

<Reveal titulo="Respuesta">
`for` clásico controla explícitamente un índice/contador. `for-each` recorre cada elemento de una colección o array directamente, sin manejar el índice vos mismo.
</Reveal>

## Problema

Necesitás guardar el stock de varios productos: no uno, sino varios valores relacionados del mismo tipo. Declarar `int stock1, stock2, stock3...` no escala ni te deja recorrerlos con un bucle.

## Modelo mental

```
int[] stock = {15, 40, 8};

índice:    0    1    2
valor:    15   40    8
```

## Explicación sencilla

Un array es una fila de casillas del mismo tipo, numeradas desde 0. Accedés a cada casilla por su número de posición (índice).

## Explicación técnica

Un array en Java tiene **tamaño fijo**: una vez creado con `new String[3]` (o con un literal `{...}`), no podés agregar ni quitar posiciones — solo modificar el contenido de las que ya existen. El primer índice es `0`, el último es `longitud - 1` (accesible como `array.length`, sin paréntesis: es un campo, no un método).

Acceder a un índice fuera de rango (negativo, o mayor/igual a `length`) no es un error de compilación — es una excepción en **tiempo de ejecución**: `ArrayIndexOutOfBoundsException`.

Un array multidimensional (`double[][]`) es, conceptualmente, un array de arrays: cada posición del array externo contiene, a su vez, otro array.

## Cómo funciona

1. `new String[3]` reserva 3 casillas vacías (`null`), que después se llenan una por una.
2. `int[] stockPorProducto = {15, 40, 8};` crea y llena el array en una sola línea (array literal).
3. `matrizPreciosPorMes[0][0]` accede a la fila 0, columna 0 del array bidimensional.
4. El `for-each` sobre `stockPorProducto` suma cada valor sin manejar índices.

## Ejemplo mínimo

**Archivo:** `proyecto/etapa-01/ArraysDemo.java`

```java
public class ArraysDemo {
    public static void main(String[] args) {
        String[] nombresProductos = new String[3];
        nombresProductos[0] = "Teclado";
        nombresProductos[1] = "Mouse";
        nombresProductos[2] = "Monitor";

        int[] stockPorProducto = {15, 40, 8};

        for (int i = 0; i < nombresProductos.length; i++) {
            System.out.println(nombresProductos[i] + ": " + stockPorProducto[i] + " unidades");
        }

        double[][] matrizPreciosPorMes = {
            {1200.0, 1250.0, 1300.0},
            {450.0, 470.0, 460.0},
        };

        System.out.println("Precio teclado mes 1: " + matrizPreciosPorMes[0][0]);
        System.out.println("Precio mouse mes 3: " + matrizPreciosPorMes[1][2]);

        double totalStock = 0;
        for (int cantidad : stockPorProducto) {
            totalStock += cantidad;
        }
        System.out.println("Stock total: " + totalStock);
    }
}
```

Nota: la clase se llama `ArraysDemo`, no `Arrays` — `java.util.Arrays` ya existe en la biblioteca estándar, y usar ese mismo nombre para tu propia clase generaría confusión (aunque técnicamente compila si no lo importás).

## Explicación paso a paso

- `nombresProductos[i]` y `stockPorProducto[i]` comparten el mismo índice `i` en el `for`, porque están relacionados posición a posición (aunque son dos arrays separados).
- `matrizPreciosPorMes[1][2]` — primer índice elige la fila (producto), segundo índice elige la columna (mes): fila 1 es "Mouse", columna 2 es el tercer mes → `460.0`.
- El `for-each` final suma sin necesitar `stockPorProducto[i]` — directamente el valor de cada vuelta.

## Ejemplo aplicado al proyecto

Guardar `nombresProductos` y `stockPorProducto` como dos arrays paralelos (relacionados solo por índice) es exactamente el problema que la Etapa 2 (Programación Orientada a Objetos) va a resolver: en vez de dos arrays sueltos, vas a tener un array de objetos `Product`, cada uno con su propio nombre y stock juntos.

## Ejercicio trabajado

Encontrar el producto con menor stock:

```java
int indiceMenorStock = 0;
for (int i = 1; i < stockPorProducto.length; i++) {
    if (stockPorProducto[i] < stockPorProducto[indiceMenorStock]) {
        indiceMenorStock = i;
    }
}
System.out.println("Menor stock: " + nombresProductos[indiceMenorStock]);
```

## Ejercicio guiado

Calculá el precio promedio del primer producto (Teclado) a lo largo de los tres meses, usando `matrizPreciosPorMes[0]`.

<Reveal titulo="Pista 1">
`matrizPreciosPorMes[0]` por sí solo ya es un array (`double[]`) de un solo producto — podés recorrerlo con for-each.
</Reveal>

<Reveal titulo="Pista 2">
Sumá los tres valores y dividí por `matrizPreciosPorMes[0].length`.

<Reveal titulo="Solución">
```java
double suma = 0;
for (double precio : matrizPreciosPorMes[0]) {
```

## Mini reto de debugging

```java
int[] numeros = new int[3];
System.out.println(numeros[0]);
```

¿Qué imprime, si nunca asignaste ningún valor?

<Reveal titulo="Diagnóstico">
Imprime `0`. Java inicializa automáticamente los arrays de tipos primitivos numéricos en `0` (y `false` para boolean, `null` para referencias) — no quedan con "basura" como en otros lenguajes de bajo nivel.
</Reveal>

## Resumen

- Un array tiene tamaño fijo; los índices van de 0 a `length - 1`.
- `length` es un campo, no un método — sin paréntesis.
- Acceder fuera de rango lanza `ArrayIndexOutOfBoundsException` en tiempo de ejecución, no un error de compilación.
- Un array multidimensional es, conceptualmente, un array de arrays.
- Arrays de primitivos numéricos se inicializan en `0` automáticamente.

## Modelo mental final

Cuando pienses en un array, imaginá un estante con casilleros numerados de una vez y para siempre: podés cambiar qué hay en cada casillero, pero no agregar un casillero nuevo al estante.
