## Qué vas a aprender

Después de esta lección serás capaz de operar sobre variables (sumar, comparar, combinar condiciones), convertir un valor de un tipo primitivo a otro de forma consciente, y usar `var` sabiendo exactamente qué tipo infiere el compilador.

## Por qué necesitas aprenderlo

Calcular un total de pedido, decidir si hay stock suficiente, o normalizar un precio — todo eso son operadores. Y cuando mezclás `int` con `double` en una cuenta, necesitás entender cómo Java convierte los tipos, o vas a tener bugs silenciosos.

## Qué debes saber antes

Tipos primitivos, referencias y constantes con `final` (Lección 1 de esta etapa).

## Recupera lo aprendido

¿Cuál es la diferencia entre un tipo primitivo y una referencia como `String`?

<Reveal titulo="Respuesta">
El primitivo guarda el valor directamente en la variable. La referencia guarda una dirección que apunta al objeto real, que vive en otro lugar de la memoria.
</Reveal>

## Problema

Tenés `stock` como `int` y `precio` como `double`. Necesitás calcular el total de una venta (`precio * cantidad`) y decidir si queda poco stock. ¿Cómo combinás tipos distintos en una misma operación sin que el resultado te sorprenda?

## Modelo mental

```
int  +  double
  │        │
  └───┬────┘
      ▼
  Java "promueve" el int a double
  antes de operar → resultado: double
```

## Explicación sencilla

Cuando mezclás un entero y un decimal en una cuenta, Java convierte automáticamente el entero a decimal antes de operar — nunca al revés, porque eso perdería información.

## Explicación técnica

**Casting implícito (widening):** Java convierte automáticamente un tipo "más chico" a uno "más grande" cuando no hay pérdida de datos posible: `int → long → float → double`. No hace falta escribir nada.

**Casting explícito (narrowing):** convertir de un tipo "más grande" a uno "más chico" (`double → int`) puede perder información, así que Java te obliga a pedirlo explícitamente con `(tipo)`. Al hacerlo sobre un decimal, **trunca** (corta la parte decimal), no redondea.

**`var`:** le pide al compilador que infiera el tipo a partir del valor asignado. `var descripcion = "texto";` es exactamente un `String` — el tipo se decide en tiempo de compilación, no es tipado dinámico. No podés usar `var` sin inicializar la variable en la misma línea.

## Cómo funciona

1. `int restante = stock - vendidos;` — resta entre enteros, resultado entero.
2. `double total = precio * cantidad;` — `cantidad` (int) se promueve a `double` antes de multiplicar.
3. `int promedioTruncado = (int) promedio;` — casting explícito, se corta la parte decimal (7.9 → 7, no 8).
4. `var descripcion = "..."` — el compilador fija el tipo como `String` en ese mismo punto.

## Ejemplo mínimo

**Archivo:** `proyecto/etapa-01/Operadores.java`

```java
public class Operadores {
    public static void main(String[] args) {
        int stock = 15;
        int vendidos = 4;
        int restante = stock - vendidos;

        double precio = 25000.0;
        int cantidad = 3;
        double total = precio * cantidad;

        boolean stockBajo = restante < 5;
        boolean hayStock = restante != 0;

        // Casting implicito: int -> double, sin perdida de datos
        double stockComoDecimal = restante;

        // Casting explicito: double -> int, se trunca (no redondea)
        double promedio = 7.9;
        int promedioTruncado = (int) promedio;

        var descripcion = "Stock restante: " + restante;

        System.out.println("Restante: " + restante);
        System.out.println("Total venta: " + total);
        System.out.println("Stock bajo: " + stockBajo);
        System.out.println("Hay stock: " + hayStock);
        System.out.println("Stock como decimal: " + stockComoDecimal);
        System.out.println("Promedio truncado: " + promedioTruncado);
        System.out.println(descripcion);
    }
}
```

`promedio` vale `7.9`. ¿Qué imprime `(int) promedio`: `7`, `8`, o un error de compilación?

```

Stock bajo: false
```

## Explicación paso a paso

- `restante < 5` y `restante != 0` son operadores **relacionales**, devuelven `boolean`.
- `double stockComoDecimal = restante;` no necesita `(double)` porque es widening — Java lo hace solo.
- `(int) promedio` sí necesita el cast explícito porque es narrowing — y trunca: `7.9` se convierte en `7`, no se redondea a `8`.
- `var descripcion` — mirá el tipo real con el que se infiere: `String`, porque el valor a la derecha es un `String`.

## Ejemplo aplicado al proyecto

Este mismo patrón —combinar `int` (cantidades) con `double` (precios) para calcular un total— es exactamente lo que va a hacer el cálculo de total de un pedido más adelante en el curso, cuando `OrderItem` multiplique cantidad por precio unitario.

## Mini reto de debugging

```java
var contador;
contador = 0;
```

¿Por qué no compila?

## Resumen

- Los operadores relacionales (`<`, `>`, `!=`, `==`) devuelven `boolean`.
- Widening (int→double) es automático; narrowing (double→int) exige `(tipo)` explícito.
- El casting explícito sobre decimales trunca, no redondea.
- `var` infiere el tipo en tiempo de compilación — no es tipado dinámico, y exige inicialización en la misma línea.

## Modelo mental final

Cuando pienses en casting, imaginá una escalera: subir (widening) es gratis y automático; bajar (narrowing) requiere que vos mismo pongas el pie con cuidado — el `(tipo)` explícito.
