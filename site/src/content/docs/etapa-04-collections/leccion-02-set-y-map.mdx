## Qué vas a aprender

Después de esta lección serás capaz de usar `Set` cuando necesitás garantizar que no haya elementos repetidos, y `Map` cuando necesitás buscar un valor por una clave (como un código de producto) en vez de por posición.

## Por qué necesitas aprenderlo

La lección anterior terminó preguntando: si necesitás que ningún producto se repita, ¿alcanza con `List`? No — `List` permite duplicados libremente. Y buscar un producto por su código recorriendo una lista entera, uno por uno, no escala.

## Qué debes saber antes

`List`/`ArrayList` y Generics básicos (`List<Tipo>`) de la lección anterior.

## Recupera lo aprendido

¿Qué diferencia hay entre `size()` de una `List` y `length` de un array?

<Reveal titulo="Respuesta">
`size()` es un método de List; `length` es un campo de array — sintaxis distinta para un concepto similar (cantidad de elementos).
</Reveal>

## Problema

Necesitás dos cosas distintas: una colección de categorías de producto donde nunca se repita ninguna (aunque intentes agregarla dos veces), y una forma de buscar un producto por su código (`SKU-001`) sin tener que recorrer toda una lista comparando uno por uno.

## Modelo mental

```
Set<String>                          Map<String, Product>
┌─────────────┐                      ┌───────────┬─────────────┐
│ Perifericos │                      │ "SKU-001" │ Teclado...  │
│ Monitores   │  sin duplicados      │ "SKU-002" │ Mouse...    │
└─────────────┘                      └───────────┴─────────────┘
                                        clave         valor
```

## Explicación sencilla

Un `Set` es como una `List` que rechaza silenciosamente los duplicados: si agregás algo que ya está, no pasa nada, pero tampoco se agrega de nuevo. Un `Map` no tiene posiciones numeradas — tiene **claves**: le pedís el valor asociado a una clave, y te lo devuelve directo, sin recorrer nada.

## Explicación técnica

`Set<Tipo>` es una interfaz; `HashSet` es su implementación más común. Determina duplicados usando `equals()` (y `hashCode()`, que vas a ver formalmente en la Etapa 6) — dos elementos que sean `equals` entre sí cuentan como el mismo, y el segundo `add()` simplemente no hace nada.

`Map<Clave, Valor>` también es una interfaz; `HashMap` es su implementación más común. No se recorre por índice — se opera por clave:
- `put(clave, valor)` — agrega o reemplaza el valor asociado a esa clave.
- `get(clave)` — devuelve el valor asociado, o `null` si la clave no existe (no lanza excepción).
- `containsKey(clave)` — verifica si la clave existe, sin necesidad de comparar con `null` después.

## Cómo funciona

1. `categorias.add("Perifericos")` se llama dos veces con el mismo valor — el `Set` solo lo guarda una vez.
2. `inventarioPorCodigo.put("SKU-001", teclado)` asocia esa clave a ese producto.
3. `inventarioPorCodigo.get("SKU-001")` devuelve el producto directamente, sin recorrer nada.
4. `inventarioPorCodigo.get("SKU-999")` (clave inexistente) devuelve `null`, en vez de lanzar una excepción.

## Ejemplo mínimo

**Archivo:** `proyecto/etapa-04/SetMapDemo.java`

```java
import java.util.HashMap;
import java.util.HashSet;
import java.util.Map;
import java.util.Set;

public class SetMapDemo {
    public static void main(String[] args) {
        Set<String> categorias = new HashSet<>();
        categorias.add("Perifericos");
        categorias.add("Monitores");
        categorias.add("Perifericos");

        System.out.println("Cantidad de categorias: " + categorias.size());
        System.out.println("Contiene Perifericos: " + categorias.contains("Perifericos"));

        Map<String, Product> inventarioPorCodigo = new HashMap<>();
        inventarioPorCodigo.put("SKU-001", new Product("Teclado mecanico", 25000.0, 15));
        inventarioPorCodigo.put("SKU-002", new Product("Mouse inalambrico", 8000.0, 40));

        Product encontrado = inventarioPorCodigo.get("SKU-001");
        System.out.println("Producto SKU-001: " + encontrado.getNombre());

        System.out.println("Existe SKU-999: " + inventarioPorCodigo.containsKey("SKU-999"));

        Product noEncontrado = inventarioPorCodigo.get("SKU-999");
        System.out.println("Buscar SKU-999 (sin existir): " + noEncontrado);
    }
}
```

## Predice

`categorias.add("Perifericos")` se llama dos veces con el mismo texto. ¿Cuánto va a dar `categorias.size()` — 2 o 3?

<Reveal titulo="Resultado real (compilado y ejecutado)">
```
Cantidad de categorias: 2
Contiene Perifericos: true
Producto SKU-001: Teclado mecanico
Existe SKU-999: false
Buscar SKU-999 (sin existir): null
```
</Reveal>

## Explicación paso a paso

- `categorias.size()` da `2`, no `3` — el segundo `add("Perifericos")` fue ignorado silenciosamente por ser un duplicado.
- `inventarioPorCodigo.get("SKU-001")` devuelve el `Product` real, sin ningún bucle explícito en tu código.
- `get("SKU-999")` no lanza excepción: devuelve `null`, porque esa clave nunca fue puesta con `put`.
- Por eso `containsKey` es más seguro que asumir: te deja preguntar "¿existe?" antes de operar sobre el resultado de `get`.

## Resumen

- `Set` garantiza que no haya elementos duplicados (según `equals`).
- `Map` asocia claves a valores; `get`/`put`/`containsKey` son sus operaciones principales.
- `Map.get()` con una clave inexistente devuelve `null`, no lanza excepción — hay que verificar antes de usar el resultado.
- `HashSet`/`HashMap` no garantizan ningún orden particular de sus elementos.

## Modelo mental final

Cuando pienses en `Map`, imaginá un diccionario real: no buscás una palabra por "posición 47", buscás directamente por la palabra (la clave) y obtenés su definición (el valor).
